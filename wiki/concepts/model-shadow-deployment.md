# Model Shadow Deployment (模型影子部署：配对特征轴并行)

机器学习模型升级的常见难题：新模型不能在生产环境上线之前严格证明优于旧版本，但等到生产环境累积足够样本又面临"半年才能 cutover"的窘境。**配对特征轴并行**是一种用存储冗余换取在线灰度自由度的部署模式——让旧模型与新模型**同时计算、同时落盘**，下游消费方按需选择，把"模型升级"从代码事件降级为配置事件。

## 核心模式

特征表预留**两套逻辑特征轴**——每套包含一个向量字段和一个版本标签字段。列名可以按模型代际命名，也可以按业务特征轴命名；关键是两套字段在 schema 中并存：

```
old feature vector + old model version
new feature vector + new model version
```

新旧模型同时跑、同时写入各自的列，每行特征因此带**双特征 + 双版本血缘**。下游消费方可以：

- 只读旧特征轴（保守路径）；
- 只读新特征轴（实验路径）；
- 同时读两路，做对照分析。

cutover 是"修改下游配置"，不是数据迁移；回滚也是修改配置，秒级生效。在 trajex 里，这个模式落成旧 ReID 特征轴与新统一特征轴并写，而不是强制使用字面 `v1/v2` 列名。^[[[_living/AI-Applications-and-Ops/ReID-Perception-Layer-trajex|ReID-Perception-Layer-trajex]]]

## 与几种相邻模式的关系

| 模式                   | 决策粒度         | 回滚成本                             | 历史数据可重算         |
| ---------------------- | ---------------- | ------------------------------------ | ---------------------- |
| **配对特征轴影子部署** | 全表统一 cutover | 改下游配置，秒级                     | 是（历史行已经双写）   |
| **请求级 A/B**         | 按请求采样       | 改流量分配                           | 否（一条请求只走一边） |
| **单列 + 重算**        | 全表统一 cutover | 需要把整列重算回旧模型，分钟到小时级 | 是，但要再跑一遍模型   |
| **双部署 + 旁路评测**  | 流量镜像         | 仅改路由                             | 否（只评测不落库）     |

影子部署相对其他模式的关键差异：**"全量历史数据双轨可比"**——下游可以在任意时间窗用新特征轴跑出结果，与旧特征轴历史对照，不必等待新窗口积累。

## 适用场景

- **下游聚合系统不可改**或**改起来慢**——[[hidalgo]] 这种离线批跑一次几小时，不可能在批跑过程中切模型；影子特征轴让聚合系统按自己节奏挑选；
- **新模型缺乏严格离线评测**——学术 SOTA 不等于业务可用，需要让新模型在生产数据上跑一段时间做对照；
- **数据库列宽度可容忍冗余**——两套 ReID 向量会显著增加行宽，适合有分区与保留期控制的宽表；
- **模型升级周期 < 数据保留期**——影子期不应过长，否则双写成本太高，需要明确的 sunset 计划。

## 与版本血缘列的协同

仅有双向量不够。每行还应该带：

- 模型版本：精确到训练 commit 或权重摘要；
- 数据来源：标记这行特征来自实时流、批回放、bug 修复重跑等不同来源——便于下游分析时按来源漏斗、debug 时定位是哪一波回放污染了数据。

把"哪个模型在哪一天通过哪种链路产生了这个向量"用列直接讲清楚——历史数据可解释、可重算、可问责。

## 边界与代价

- **存储翻倍**：双特征列 + 索引开销，按天分区控制总量；
- **推理算力翻倍**：旧模型与新模型同时跑，GPU 资源消耗双倍；通常用异步并发让两路模型共享同一批前处理结果；
- **不适合极频繁迭代的模型**：如果一个月迭代一次，新增 v3、v4、v5 列会让表结构膨胀失控。该模式更适合**主干升级 + 小步微调**的中等迭代节奏；
- **下游必须显式声明读哪一列**——这是把"模型版本"提升为下游 API 显式参数的代价，避免隐式切换带来的 silent regression。

ReID 特征模型选型见 [[reid-embedding-models]]——影子部署是该谱系下"如何从 FastReid 平滑过渡到 SOLIDER / CLIP-ReID"的具体落地模式。
