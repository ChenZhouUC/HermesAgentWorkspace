# Customer Flow Post-Processing (客流后处理加工)

线下零售门店 [[reid-pipeline|ReID 系统]] 的下游处理层：把行人 ID + 轨迹列表加工为业务指标（客流量、批次、接待）。这是 ReID 原子结果与业务消费之间的"翻译层"，独立部署、独立调参，可与 ReID 核心解耦做差异化策略。

> **代码边界说明**：本概念描述的接待识别 / 多信号过滤策略**不在 ReID 核心引擎里**——它们运行在独立的下游服务中（不同的代码仓库与部署单元）。本页方法学来自团队内部的设计与测试报告，confidence 标为 medium。

## 与 ReID 核心解耦

ReID 输出的是 _去重后的行人 ID + 角色标签 + 轨迹列表_；它**不产出业务指标**。后处理把这些原始结果加工为：

- **客流量**：进店人数、批次数、人均停留时长；
- **接待数据**：每条顾客轨迹是否被店员接待、接待开始时间、接待时长；
- **属性聚合**：年龄、性别、进车动作等业务标签。

这种分层让 ReID 模型升级与业务策略迭代互不阻塞，业务侧也能按租户/门店独立灰度新策略而不影响其他门店。^[[[_living/AI-Applications-and-Ops/Customer-Flow-Post-Processing|Customer-Flow-Post-Processing]]]

## 接待识别：多维评分 + 状态机时间窗

判断"某段时间内顾客是否被某店员接待"是一个典型的人体交互识别问题。坐标映射方案误差大，更稳的做法是直接在**同一摄像头原始坐标系**内对比顾客与店员的轨迹点。

### 1. 轨迹特征预处理

每个轨迹点计算两组基础量：

- **速度**：相邻点欧氏距离 / 时间差，做指数移动平均平滑以抑制点位抖动；
- **方向**：相邻点位移单位向量。

### 2. 单点多维加权评分

对每个顾客点，在一个对称时间窗内匹配所有店员点，单点分综合以下几类正交信号：

1. 两人距离（超过经验阈值直接 0 分，硬截断避免远距离误关联）；
2. 顾客是否朝店员走；
3. 顾客是否停下；
4. 双方是否同时停下；
5. 两人运动方向是否一致；
6. 近距离 + 双方都停的额外加分。

设计要点：把"距离 + 朝向 + 停留 + 一致性"加权融合，而不是只看距离——单一信号容易被对抗样本穿透。

### 3. 轨迹分聚合 + 规则化保底

按顾客点逐点取最优店员点评分后聚合成轨迹分，再用一组"由强到弱"的规则逐级 _取 max_ 保底：

- **强交互**：单点出现明显高分时直接给出高保底；
- **中等累积**：相当比例的点处于中等分且均分达标时给出中等保底；
- **低分累积**：少量低分点广泛分布时给出弱保底；
- **松散跟随**：长轨迹且大比例点维持微弱信号时给出兜底保底。

遍历所有店员取分最高者作为最终分。**关键设计**：单点高分（强交互）与长程低分累积（松散跟随）走两条不同的提分路径，互不冲突——避免长接待因为没出现某一秒"高光时刻"被漏判。

### 4. 接待时间窗的状态机提取

当轨迹分超过弱接待阈值时，对最佳店员做点序时间评分，用状态机抽取接待段：

- **进入**：点分超过进入阈值；
- **维持**：点分高于维持阈值即可继续（低分跟随不打断）；
- **退出**：点分低于维持阈值且持续一段时间。

多段窗口按短间隔合并；有效接待时长 = 合并后所有窗口时长之和（无窗口时回退到顾客全程时长）。

### 5. 不完整轨迹兜底

顾客轨迹经常因遮挡或检测丢失而残缺。当主判定为"未接待"时触发兜底：

- **触发条件**：主判定为未接待 + 兜底开关已开 + 轨迹被判为不完整（独立轨迹段数过少 / 总点数过少 / 最大时间间隔过大，任一满足）；
- **兜底判定**：计算顾客与所有店员（不限摄像头）的时间区间交集，最大交集比例超过阈值即改判为接待，并把得分补到一个最低水位。

这种兜底承认了"主算法对不完整轨迹无能为力"，转而用粗粒度的时间重叠做最后判定，宁可接受一定误判也不漏判。

## 零售场景的客流过滤模式

ReID 输出会包含若干典型的"伪进店"轨迹，必须在后处理层剔除。**分工边界**：其中"小门框过店"过滤已经在 [[reid-pipeline|ReID 核心引擎]] 里做了一道（基于轨迹的像素跨度 + 单调方向检测）；本概念描述的"假人 / 嵌套检测"等过滤是后处理层叠加的补强。

- **静态物体伪进店**：服装/汽车门店常把假人或样品摆在出入口附近，检测框抖动或人工搬运会生成伪轨迹。判别特征是 bbox 极小且稳定 + 行进方向缺乏一致矢量。
- **嵌套检测伪人**：特殊发型让边缘端模型把头肩误识为人体、发型部位误识为头部，产生"人套娃"双重检测框。判别特征是上衣置信度偏低 + 检测框存在嵌套关系。
- **过店误识进店**：进店区域过小、门框较窄的门店，过店行人会被误判为进店。判别特征是轨迹长度过短、停留点少、与店内深度区域不相交。

**通用原则**：依赖单一检测信号容易被对抗样本（假人、嵌套、过店）穿透；多信号交叉验证（bbox + 方向 + 置信度 + 几何关系）是更鲁棒的过滤范式。

## 配置化设计

后处理参数按店/按租户落地，按需开启而不动代码：

- 小孩年龄过滤阈值；
- 进店区域过小的店开启进店轨迹过滤；
- 人次过滤（基于 ReID 结果决定重复进店是否过滤、超过多少次后过滤、过滤方式）；
- 接待兜底开关与模式选择；
- 聚合一致性阈值（默认关闭，特殊店启用）。

配置表与代码解耦，按租户/门店灰度新策略而不影响其他门店——这是把"实验性策略"安全推进生产的关键基础设施。同样的"按店参数化"思想在 [[reid-pipeline|ReID 核心引擎]] 也有对偶实现（按店覆写默认聚类参数），上下游共用同一套运维模式。
