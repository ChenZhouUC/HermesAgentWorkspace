# 客流后处理加工逻辑

本文记录 ReID 系统下游的后处理逻辑：把行人 ID + 轨迹结果加工为业务指标（客流量、批次、接待）。
ReID 系统本体见 `ReID-Pipeline-Architecture.md`；本文不重复 ReID 内部链路。

> **代码边界说明**：hidalgo 计算层服务（Hidalgo 也常作为整个 ReID 项目名使用）只产出"去重后的行人 ID + 角色 + 轨迹"，**不包含本文描述的接待识别与客流过滤逻辑**。这些后处理模块运行在一个独立的下游服务里（与 ReID 核心是不同的代码仓库与部署单元）。本文记载的是该下游服务的方法学，源自团队内部的设计与测试报告。

## 1. 后处理职责范围

ReID 输出的是 _去重后的行人 ID + 角色标签 + 轨迹列表_；它本身**不产出业务指标**。后处理把这些原始结果加工为：

- **客流量**：进店人数、批次数、人均停留时长；
- **接待数据**：每条顾客轨迹是否被店员接待、接待开始时间、接待时长；
- **属性聚合**：年龄、性别、进车动作等业务标签。

后处理逻辑独立部署、独立调参，与 ReID 核心解耦，便于按租户/门店做差异化策略。

## 2. 接待识别

判断"某段时间内顾客是否被某店员接待"是一个典型的人体交互识别问题。坐标映射方案误差大，更稳的做法是直接在**同一摄像头原始坐标系**内对比顾客与店员的轨迹点。

### 2.1 轨迹特征预处理

每个轨迹点计算两组基础量：

- **速度**：相邻点欧氏距离 / 时间差，做指数移动平均平滑以抑制点位抖动；
- **方向**：相邻点位移单位向量。

### 2.2 单点多维加权评分

对每个顾客点，在一个对称时间窗内匹配所有店员点，单点分综合以下几类正交信号：

1. 两人距离（超过经验阈值直接 0 分，硬截断避免远距离误关联）；
2. 顾客是否朝店员走；
3. 顾客是否停下；
4. 双方是否同时停下；
5. 两人运动方向是否一致；
6. 近距离 + 双方都停的额外加分。

设计要点：把"距离 + 朝向 + 停留 + 一致性"加权融合，而不是只看距离——单一信号容易被对抗样本穿透。

### 2.3 轨迹分聚合 + 规则化保底

按顾客点逐点取最优店员点评分后聚合成轨迹分，再用一组"由强到弱"的规则逐级 _取 max_ 保底：

- **强交互**：单点出现明显高分时直接给出高保底；
- **中等累积**：相当比例的点处于中等分且均分达标时给出中等保底；
- **低分累积**：少量低分点广泛分布时给出弱保底；
- **松散跟随**：长轨迹且大比例点维持微弱信号时给出兜底保底。

遍历所有店员取分最高者作为最终分。**关键设计**：单点高分（强交互）与长程低分累积（松散跟随）走两条不同的提分路径，互不冲突——避免长接待因为没出现某一秒"高光时刻"被漏判。

### 2.4 接待时间窗的状态机提取

当轨迹分超过弱接待阈值时，对最佳店员做点序时间评分，用状态机抽取接待段：

- **进入**：点分超过进入阈值；
- **维持**：点分高于维持阈值即可继续（低分跟随不打断）；
- **退出**：点分低于维持阈值且持续一段时间。

多段窗口按短间隔合并；有效接待时长 = 合并后所有窗口时长之和（无窗口时回退到顾客全程时长）。

### 2.5 不完整轨迹兜底

顾客轨迹经常因遮挡或检测丢失而残缺。当主判定为"未接待"时触发兜底：

**触发条件**（全部满足）：

- 主判定为"未接待"；
- 该门店的兜底开关已开启（严格模式下直接跳过兜底）；
- 轨迹被判为不完整（独立轨迹段数过少 / 总点数过少 / 最大时间间隔过大，任一满足）。

**兜底判定**：计算顾客与所有店员（不限摄像头）的时间区间交集，最大交集比例超过阈值即改判为接待，并把得分补到一个最低水位。

这种兜底承认了"主算法对不完整轨迹无能为力"，转而用粗粒度的时间重叠做最后判定，宁可接受一定误判也不漏判。

### 2.6 上线效果

灰度对比显示：

- 平均接待率变化在小幅范围内；
- 上升门店与下降门店数量大致相当，证明逻辑不偏向某一侧。

主要 false negative 来源：店员长时间坐着不动导致店员轨迹消失。需要在 ReID 侧补全店员库的"店内静态轨迹"样本。

## 3. 客流过滤策略

ReID 输出会包含若干典型的"伪进店"轨迹，必须在后处理层剔除。

**分工边界**：其中"小门框过店"判别已经在 hidalgo 计算层内做了一道（基于轨迹的像素跨度 + 单调方向检测，把单纯过店改判为 invalid 轨迹）；本文描述的"假人 / 嵌套检测"等过滤是后处理层叠加的补强。

### 3.1 静态物体伪进店（假人 / 样品）

服装/汽车门店常把假人或样品摆在出入口附近。检测框抖动 + 人工搬运会产生伪进店轨迹。
判别特征：

- **外包围框尺寸**：假人轨迹的 bbox 极小或异常稳定；
- **轨迹行进方向**：真人有明确的进/出方向矢量，搬运假人方向随机。

### 3.2 嵌套检测伪人（特殊发型）

特殊发型让边缘端模型把头肩误识为人体、发型部位误识为头部，产生"人套娃"双重检测框。
判别特征：

- **上衣置信度分值**：套娃情况下分数明显偏低；
- **检测框相对人体的位置**：嵌套关系可识别。

### 3.3 过店误识进店（小门框场景）

进店区域过小、门框较窄的门店，过店行人会被误判为进店。
策略：在轨迹层加进店轨迹过滤条件（轨迹像素跨度、停留点位、与店内深度区域是否相交），把单纯过店从进店中剔除。该策略已在 hidalgo 计算层实现，但默认参数偏保守，按店调整。

**通用原则**：依赖单一检测信号容易被对抗样本（假人、嵌套、过店）穿透；多信号交叉验证（bbox + 方向 + 置信度 + 几何关系）是更鲁棒的过滤范式。

## 4. 配置化设计

后处理参数按店/按租户落地，按需开启而不动代码：

- 小孩年龄过滤阈值；
- 进店区域过小的店开启进店轨迹过滤；
- 人次过滤（基于 ReID 结果决定重复进店是否过滤、超过多少次后过滤、过滤方式）；
- 接待兜底开关与模式选择；
- 聚合一致性阈值（默认关闭，特殊店启用）。

配置表与代码解耦，按租户/门店灰度新策略而不影响其他门店——这是把"实验性策略"安全推进生产的关键基础设施。同样的配置覆写思想在 hidalgo 计算层也有对偶实现（按店覆写默认参数），上下游共用同一套"按店参数化"的运维模式。
